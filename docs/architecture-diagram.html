3. Przepływ danych — Wiadomość od wysłania do AI

Pełny cykl życia wiadomości i jak wpływa na system

%%{init: {'theme': 'dark', 'themeVariables': { 'primaryColor': '#22d3ee', 'primaryTextColor': '#e2e8f0', 'lineColor': '#475569', 'secondaryColor': '#1e1b4b', 'tertiaryColor': '#1a1a2e', 'fontSize': '13px'}}}%%
sequenceDiagram
    participant U as 👤 User
    participant FE as 🖥️ Frontend
(Next.js)
    participant CV as ⚡ Convex
(Backend)
    participant DB as 💾 Convex DB
    participant AI as 🤖 OpenAI
(GPT-4o)
    participant KG as 🧠 Graphiti
(Knowledge Graph)

    Note over U,KG: PRZEPŁYW 1: Wysyłanie wiadomości
    U->>FE: Pisze wiadomość w kanale
    FE->>CV: messages.send(body, channelId)
    CV->>DB: INSERT message
    DB-->>FE: Real-time WebSocket update
    FE-->>U: Wiadomość pojawia się natychmiast

    Note over U,KG: PRZEPŁYW 2: Ingest do Knowledge Graph
    CV->>KG: POST /ingest (message data)
    KG->>KG: Ekstrakcja encji i relacji
    KG-->>CV: OK (episode_id)
    CV->>DB: UPDATE message.graphitiEpisodeId

    Note over U,KG: PRZEPŁYW 3: AI Summarization (CRON co 15 min)
    CV->>DB: Pobierz nowe wiadomości z kanałów
    DB-->>CV: Messages since last summary
    CV->>AI: Prompt: "Podsumuj w 3 bulletach"
    AI-->>CV: {bullets, actionItems, priorities}
    CV->>DB: UPSERT inboxSummaries (per user)
    DB-->>FE: Real-time update → Inbox
    FE-->>U: Nowe podsumowanie w Inbox

    Note over U,KG: PRZEPŁYW 4: @KnowledgeBot Q&A
    U->>FE: "@KnowledgeBot jak zdecydowaliśmy o X?"
    FE->>CV: messages.send (wykrywa @mention)
    CV->>KG: POST /query (question)
    KG-->>CV: Ranked results + citations
    CV->>AI: Prompt + context → cited answer
    AI-->>CV: Answer with source references
    CV->>DB: INSERT bot message (type=bot)
    DB-->>FE: Real-time update
    FE-->>U: Bot odpowiada z cytowaniami

4. Przepływ autentykacji

Od logowania do autoryzacji zapytań

%%{init: {'theme': 'dark', 'themeVariables': { 'primaryColor': '#fbbf24', 'primaryTextColor': '#e2e8f0', 'lineColor': '#475569', 'secondaryColor': '#1e1b4b', 'tertiaryColor': '#1a1a2e', 'fontSize': '13px'}}}%%
sequenceDiagram
    participant U as 👤 User
    participant FE as 🖥️ Next.js
    participant MW as 🛡️ Middleware
    participant WOS as 🔑 WorkOS
    participant CV as ⚡ Convex

    U->>FE: Odwiedza /inbox
    FE->>MW: Sprawdza sesję
    MW-->>FE: Brak sesji → redirect
    FE-->>U: Przekierowanie na /login

    U->>FE: Klika "Login with Google"
    FE->>WOS: OAuth Authorization Request
    WOS-->>U: Google login prompt
    U->>WOS: Loguje się (Google OAuth)
    WOS-->>FE: Redirect do /callback + auth code

    FE->>WOS: Exchange code → session + JWT
    WOS-->>FE: WorkOS session token

    FE->>FE: /api/auth/token
    FE->>CV: ConvexProviderWithAuth(token)
    CV->>CV: ctx.auth.getUserIdentity()
    CV->>CV: users.getMe(workosUserId)
    CV-->>FE: User data (real-time subscription)
    FE-->>U: Dashboard załadowany ✅
